# 🛡️ Alumni Directory - Admin Guide

This platform utilizes a powerful, protected Command Center built for securely manipulating the Firebase database. 

## 🔐 Accessing the Dashboard
Click the **Admin Access** button at the bottom of the public directory sidebar. 
*   You will be prompted for an Email and Password. 
*   If you login successfully, you will be redirected to the Split-Pane dashboard. 
*   To change your password, use the **Password** button in the sidebar. Note: Firebase requires "recent authentication" to change sensitive credentials, so you may need to log out and log back in before doing this.

---

## 💻 The Split-Pane Dashboard

The Admin Panel relies on a high-density, dual-column view:
*   **Left Pane:** The main Database Table.
*   **Right Pane:** Two Action Queues (Direct Messages & Profile Approval Requests).

### Maximizing Screen Real Estate
Because the database tables are extremely wide, working on a laptop screen can feel cramped.
*   Click the **Expand (<i class="fas fa-expand"></i>)** icon in the top right corner of either the Database Pane or the Action Queues Pane.
*   This instantly sweeps the opposite pane off-screen and expands your active workspace to 100% width, automatically hiding the sidebar to prevent visual distraction.
*   Click the **Compress (<i class="fas fa-compress"></i>)** icon to restore the dual-pane view. 
*   *Note: On mobile devices, these panes automatically act as toggleable Tabs so you never have to endlessly scroll.*

---

## 🗄️ Database Management

### Adding a New Record Manually
Click **Add Record** at the top of the Database pane. This opens the master Editor Modal.
1.  **Batch Routing:** Selecting a Programme and Start Year automatically calculates the user's batch string (e.g., `PhD 2020`).
2.  **Avatar Upload:** The file picker securely uploads images to Cloudinary. It automatically enforces a **1MB size limit** to preserve network bandwidth and prevent spam uploads.
3.  Click **Commit Save** to write it to Firebase.

### Editing & Deleting
Each row in the database table has action buttons attached to the far right.
*   **Edit (<i class="fas fa-edit"></i>):** Re-opens the Editor Modal pre-filled with the student's existing data! If you upload a new photo, it automatically overwrites the old link.
*   **Delete (<i class="fas fa-trash"></i>):** Permanently purges the record from the Firebase cloud.

---

## 📩 Action Queues

The right-hand pane handles requests natively generated by public users.

### 1. Update Requests (Approvals)
Whenever a user fills out the "Join" or "Update" forms on the public site, their data appears here.
*   **Green "New Join" Badge:** Means the user doesn't exist yet. Clicking "Approve" will create a brand new profile.
*   **Yellow "Update Request" Badge:** Means the user has submitted changes to their existing data. Clicking "Approve" will safely **merge and overwrite** their active profile without creating duplicates.
*   Clicking **Reject (<i class="fas fa-times"></i>)** permanently deletes their request. 

### 2. Support Messages (Inbox)
Any communication sent via the "DM" module appears here chronologically. These do not contain database profiles—just text notes. Click the 'X' button to dismiss them once addressed.
